# AI+搜索 对话推荐模块

## 功能目标
在首页顶部 Banner 区放置一个自然语言输入框。
定位为“AI+搜索”的统一入口。用户可以输入明确的工具名称（如“Gamma”），也可以输入自然语言需求（如“我想做 PPT 用什么 AI”）。系统调用 AI 接口，理解意图并返回 1~3 个最合适的工具，以结构化卡片形式展示。

这是 Demo 阶段唯一需要实现的"智能交互"能力。

## 用户交互流程
```
用户输入问题
→ 点击发送或按回车
→ 前端调用 /api/recommend
→ 服务端拼接固定 Prompt + 候选工具列表
→ 调用 DeepSeek 或 Claude API
→ 返回结构化 JSON
→ 前端解析 JSON
→ 页面渲染工具推荐卡片
→ 用户点击卡片进入工具详情页
```

### 示例输入
- 我想做 PPT 用什么 AI
- 写公众号文章推荐什么工具
- 我想搭一个企业知识库助手
- 帮我找一下 Gamma 这个工具 （直接搜索工具）
- 做数据分析有哪些 AI 可用

## 输入区设计要求
首页 AI 对话输入区必须满足以下要求：
- 一个文本输入框
- 一个发送按钮
- 支持回车提交
- 支持按钮点击提交
- 输入为空时不可提交
- 自动去除前后空格
- 输入长度限制为 100 到 150 字
- 请求发送后按钮进入 loading 状态
- 请求完成前禁止重复提交

占位文案建议为：
**输入 AI 工具名称，或描述你的需求（如：做 PPT 用什么 AI？）**

按钮文案建议：
- 默认：智能搜索
- 加载中：推荐中...

## 输出区设计要求
推荐结果展示在输入框下方。

每张推荐卡片必须包含：
- 工具名称
- 推荐理由
- 标签列表
- 查看详情按钮

展示形式建议为横向卡片布局：
- PC 端每行展示 2 到 3 张
- 移动端自动换行
- 卡片样式统一、简洁、有科技感

点击"查看详情"后跳转到站内详情页，例如：
`/tools/gamma`

## Prompt 规则
服务端调用 AI 时必须使用固定 Prompt。AI 需要判断用户是“精准搜索某个工具”还是“描述场景需求”，并从站内已有工具列表中进行匹配和推荐，不能推荐站外未收录工具。

### System Prompt
```
你是一个 AI 工具搜索与推荐助手。

用户会输入搜索词或描述自己的需求。你需要理解用户的意图：
1. 如果用户是直接搜索特定工具（如输入"Gamma"或"有没有类似Canva的"），请直接返回该工具或平替工具的信息。
2. 如果用户是描述场景需求，请从候选工具列表中推荐 1~3 个最合适的 AI 工具。

候选工具只能从以下列表中选择，不允许输出列表外工具：
- Gamma
- Beautiful.ai
- Tome
- Canva
- ChatGPT
- Claude
- DeepSeek
- 豆包
- 通义千问
- Coze

每个工具必须返回以下字段：
- name：工具名称
- slug：用于跳转详情页的唯一标识
- reason：推荐理由，20~36字以内
- tags：标签数组，1~3个，每个标签不超过8个字
- url：官网地址

只返回 JSON 数组，不要返回 markdown，不要返回解释，不要返回任何其他内容。
```

## AI 推荐结果字段定义
每个工具对象必须包含以下字段：

- `name`：工具名称
- `slug`：详情页唯一标识
- `reason`：推荐理由
- `tags`：标签数组
- `url`：官网地址

注意：
- `reason` 长度控制在 20~36 字以内
- `tags` 数量 1~3 个
- 每个 tag 不超过 8 个字
- 推荐数量控制在 1~3 个之间

## 异常处理要求

### 1）接口请求失败
展示提示：
**暂时无法获取推荐，请稍后重试**

### 2）返回内容无法解析为 JSON
展示提示：
**推荐结果解析失败，请稍后重试**

### 3）返回空数组
展示提示：
**暂未找到合适工具，试试换个问法**

### 4）接口超时
服务端建议设置 10 秒超时。
超时提示：
**推荐生成超时，请稍后重试**
