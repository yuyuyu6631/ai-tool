# 数据结构规范

## 说明
Demo 阶段使用静态 mock 数据即可，不需要真实数据库。

建议准备 12 到 20 个工具的数据，用于支撑首页、榜单页、详情页和 AI 对话推荐。

## 工具数据字段
每个工具最少需要以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| slug | string | 唯一标识，用于路由 |
| name | string | 工具名称 |
| category | string | 分类 |
| score | number | 综合评分 |
| summary | string | 一句话简介 |
| tags | string[] | 标签数组 |
| pros | string[] | 优点列表 |
| cons | string[] | 缺点列表 |
| scenarios | string[] | 适用场景 |
| alternatives | string[] | 平替工具 slug 列表 |
| officialUrl | string | 官网地址 |
| editorComment | string | 编辑点评 |

## 示例数据
```json
{
  "slug": "gamma",
  "name": "Gamma",
  "category": "PPT生成",
  "score": 9.2,
  "summary": "适合快速生成演示文稿和视觉化内容。",
  "tags": ["PPT", "演示", "生成"],
  "pros": ["生成速度快", "适合演示场景"],
  "cons": ["高级模板部分受限", "细节可控性一般"],
  "scenarios": ["做汇报PPT", "快速提案"],
  "alternatives": ["beautiful-ai", "canva"],
  "officialUrl": "https://gamma.app",
  "editorComment": "适合希望快速出稿的用户，尤其适合对排版效率有要求的场景。"
}
```
