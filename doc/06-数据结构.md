# 06 数据结构

## 1. 当前实现说明

当前项目已经以真实数据库模型作为系统基线。  
前端仍保留 `apps/web/src/data/*.json` 作为仓库内资产，但公开页面的主数据来源已经是 API。

## 4. 前端关键类型

### 4.1 工具类型

- `ToolSummary`
- `ToolDetail`

`ToolSummary` 当前已包含：

- 基础展示字段（id / slug / name / category / score / summary / tags 等）
- `reviewCount`
- `accessFlags`
- `pricingType`
- `priceMinCny`
- `priceMaxCny`
- `freeAllowanceText`
- `features`
- `limitations`
- `bestFor`
- `dealSummary`
- `primaryMedia`
- `reason`

`ToolDetail` 在此基础上继续包含：

- `targetAudience`
- `abilities`
- `pros`
- `cons`
- `pitfalls`
- `scenarios`
- `scenarioRecommendations`
- `reviewPreview`
- `ratingSummary`
- `mediaItems`
- `alternatives`

### 4.2 评论类型

- `ToolReviewAuthor`
- `ToolReviewItem`
- `ToolRatingSummary`
- `ToolReviewsResponse`
- `UpsertToolReviewRequest`

### 4.3 后台类型

- `AdminToolPayload`
- `AdminToolListItem`（含 `featured`、`score`、`reviewCount`、`updatedAt`）
- `AdminOverviewResponse`
- `AdminReviewListItem`（含 `toolId`、`status`、`rating`、`createdAt`、`updatedAt`）
- `AdminRankingPayload`
- `AdminRankingListItem`（含 `description`）

### 4.4 认证类型

- `AuthUser`

当前字段包括：

- `id`
- `username`
- `email`
- `status`
- `role`
- `createdAt`

## 5. 当前数据边界

- 后端 schema 以 `apps/api/app/schemas/*.py` 为准
- 前端本地类型位于 `apps/web/src/app/lib/catalog-types.ts` 与 `apps/web/src/app/lib/auth-api.ts`
- `packages/contracts` 仍是共享契约收敛点，但前端尚未完全切换到共享包直引
