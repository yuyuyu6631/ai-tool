# 前端落地说明

本文件用于把“设计意图”转换成“当前代码已落地的前端实现说明”。它不是纯视觉稿，而是对现有页面结构、设计 token、交互状态和组件落地方式的汇总。

## 1. 当前前端技术栈定稿

### 1.1 主框架

- 当前实际采用：`React 18`
- 路由方案：`React Router 7`
- 构建工具：`Vite 6`

结论：

- 当前项目不是 Next.js 项目。
- 若后续需要 SEO、服务端渲染或内容静态化，再单独评估迁移，不影响当前 Demo 的交互演示目标。

### 1.2 CSS 方案

- 当前实际采用：`Tailwind CSS 4`
- 辅助方式：`theme.css` 中的 CSS Variables + 语义化组件类名

这意味着当前风格不是单纯“原子类堆叠”，而是：

- 主题 token 统一维护
- `card-base`、`panel-base`、`btn-primary` 等样式统一沉淀
- 页面层只补充布局和响应式 utility

## 2. 视觉参数定稿

以下参数以当前代码为准。

### 2.1 色彩

| Token | 值 | 用途 |
| --- | --- | --- |
| `--background` | `#eef4fb` | 页面天空感主背景 |
| `--foreground` | `#0f172a` | 主文字 |
| `--card` | `rgba(255, 255, 255, 0.5)` | 玻璃卡片基础底 |
| `--primary` | `#11203b` | 主按钮背景 |
| `--secondary` | `rgba(255, 255, 255, 0.2)` | 次级玻璃底色 |
| `--muted` | `rgba(255, 255, 255, 0.22)` | 弱化区块背景 |
| `--muted-foreground` | `#4f5f78` | 次级文字 |
| `--accent-foreground` | `#171717` | 强调区内容色 |
| `--destructive` | `#d4183d` | 错误态 |

当前视觉方向：

- 天空蓝渐层背景
- 玻璃拟态卡片与面板
- 白色半透明层次为主
- 用暖橙与天蓝光斑增加氛围

### 2.2 圆角

| 场景 | 当前值 |
| --- | --- |
| 全局基础圆角变量 | `1rem` |
| 大卡片常用圆角 | `rounded-[26px]` 到 `rounded-[28px]` |
| 搜索面板圆角 | `24px` |
| 输入框与按钮 | `rounded-2xl` 或 `rounded-full` |
| Logo/图标 tile | `rounded-xl` / `rounded-2xl` |

### 2.3 阴影与边框

当前方案改为轻玻璃风：

- 卡片基础阴影：`0 18px 40px rgba(15, 23, 42, 0.08)`
- Hero 面板阴影：`0 30px 80px rgba(15, 23, 42, 0.16)`
- 统一使用半透明白边框 `rgba(255, 255, 255, 0.18 ~ 0.28)`
- 广泛使用 `backdrop-filter: blur(...) saturate(...)`

### 2.4 排版

当前代码中已形成如下层级：

- 首页主标题：`text-4xl` 到 `md:text-[3.8rem]`
- 页面标题：`text-3xl` 到 `md:text-4xl`
- 区块标题：`text-2xl` 到 `md:text-3xl`
- 卡片标题：`text-lg` 或 `text-[17px]`
- 主正文：`text-sm` 到 `text-lg`

### 2.5 动效

当前动效策略是轻量氛围型：

- 路由顶部反馈条
- 卡片 hover 上浮并轻微放大
- 按钮 hover 上浮 `2px`
- `prefers-reduced-motion` 已做兼容

## 6. 当前与原始需求的偏差说明

### 6.1 偏差一：技术栈

- 原讨论中存在 “Vue 3 / Next.js 二选一” 的待定状态
- 当前代码已经落到 `React + Vite`

### 6.2 偏差二：设计方向

- 原需求里提到过 Bento Grid、彩色渐变风
- 当前实现实际落到了“轻玻璃工作台 + iOS 小组件感”的首页方向
- 但整体信息架构与工具目录可读性仍然保留

### 6.3 偏差三：假数据准备度

- 原需求里担心缺少 mock data
- 当前代码已经内置工具与场景数据，不需要从零开始补结构

## 7. 后续前端优化建议

1. 补齐首页与其他页面之间的风格衔接，避免只有首页玻璃感明显。
2. 清理未实际使用的预置 UI 依赖与组件，降低包体和认知负担。
3. 为 `tools.json`、`scenarios.json` 做一次合法 JSON 清洗，方便程序化统计和后续扩容。
4. 将外部 Logo 图包与工具 slug 建立映射，补齐卡片视觉识别度。
5. 若后续要做“知识库升级”，建议优先把推荐逻辑从关键词规则迁移为配置驱动或服务端接口。
