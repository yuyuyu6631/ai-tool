# UI / UX 设计规范

## 总体视觉方向
整体风格必须符合以下关键词：

- 简洁
- 现代
- 克制
- 稳定
- 平台感
- 轻内容产品风格

**不要**做成传统企业官网，不要过度炫技，也不要出现明显"AI 模板页"气质。

## 推荐参考风格
设计可参考以下方向，但不要直接照搬：

- Linear / Vercel 式克制感
- Notion 式整洁排版
- Stripe / Apple 官网的节奏控制
- 内容产品常见的信息优先级表达

## 页面表现要求
- 首屏要有冲击力
- 首屏不能过满
- 卡片不能太密、太碎、太平均
- 标题层级清楚
- 标签、评分、按钮视觉统一
- 主色控制在 2 到 3 个以内
- 适合快速浏览
- 看上去像"已经有一定基础的平台"

额外要求：

- 基于 Apple-style 干净白底 + 玻璃拟态装饰构建平台感风格
- 通过毛玻璃效果、留白、比例、字重、对齐共同体现审美
- 不依赖过度炫技（不使用大面积渐变、刺眼发光效果）
- 每个区块都要有明确主次，不能同等强调

## 已落地具体设计规范

### 色彩体系

当前采用 CSS 自定义属性（`theme.css :root`）管理全站色彩，支持 **亮色 / 暗色** 双主题切换。

#### 亮色模式（`:root`）

| 类别 | 色值/取值 | 说明 |
|------|-----------|------|
| 页面背景 | `--background: #ffffff` | 干净白底 |
| 页面壳背景 | `.page-shell` → `#f7f8fb` | 浅灰蓝底色 |
| 前景色 | `--foreground: #1d1d1f` | 近黑色主文字 |
| 主色 | `--primary: #0066cc` | 蓝色，用于主操作和强调 |
| 主色文字 | `--primary-foreground: #ffffff` | 主色按钮上的白色文字 |
| 次要背景 | `--secondary: rgba(0, 0, 0, 0.04)` | 极淡灰 |
| 弱化文字 | `--muted-foreground: #86868b` | 辅助信息、元数据 |
| 强调底色 | `--accent: rgba(0, 102, 204, 0.1)` | 主色淡化版，用于高亮 |
| 危险色 | `--destructive: #ff3b30` | 删除、错误状态 |
| 边框 | `--border: rgba(0, 0, 0, 0.1)` | 全局默认边框 |
| 卡片背景 | `--card: rgba(255, 255, 255, 0.72)` | 半透明白底 |
| 输入框背景 | `--input-background: rgba(255, 255, 255, 0.8)` | 轻微半透 |
| 圆角基准 | `--radius: 1rem`（16px） | 全站圆角基数 |

#### 暗色模式（`.dark`）

| 类别 | 色值/取值 |
|------|-----------|
| 背景 | `#000000` |
| 前景 | `#f5f5f7` |
| 主色 | `#0a84ff` |
| 卡片 | `rgba(28, 28, 30, 0.72)` |
| 边框 | `rgba(255, 255, 255, 0.15)` |
| 弱化文字 | `#86868b` |

#### 语义标签配色

使用 Tailwind 语义色板：

- 免费：绿色 `bg-green-100 text-green-800`
- 免费增值：蓝色 `bg-blue-100 text-blue-800`
- 订阅：紫色 `bg-purple-100 text-purple-800`
- 一次性付费：橙色 `bg-orange-100 text-orange-800`
- 已发布：翡翠绿 `bg-emerald-100 text-emerald-800`
- 草稿：琥珀黄 `bg-amber-100 text-amber-800`
- 已归档：灰色 `bg-slate-200 text-slate-700`

### 玻璃拟态核心参数

当前实际玻璃拟态参数（以 `search-panel` 为典型代表）：

```css
/* 搜索面板 */
background: rgba(255, 255, 255, 0.96);
border: 1px solid rgba(226, 232, 240, 0.84);
backdrop-filter: blur(28px) saturate(145%);
box-shadow:
  inset 0 1px 0 rgba(255, 255, 255, 0.94),
  0 24px 50px rgba(15, 23, 42, 0.055);

/* 侧边面板（aside-panel，更典型的玻璃效果） */
background: rgba(255, 255, 255, 0.16);
border: 1px solid rgba(255, 255, 255, 0.2);
backdrop-filter: blur(30px) saturate(150%);
box-shadow:
  inset 0 1px 0 rgba(255, 255, 255, 0.35),
  0 12px 30px rgba(15, 23, 42, 0.06);
```

辅助增强：使用 SVG `turbulence` 滤镜做微小扭曲，增强真实玻璃质感（应用于 `search-panel::before`、`chip-button::before` 等）。

### 排版规范

全站 base 层统一样式（`@layer base`）：

| 元素 | 字号来源 | 字重 | 行高 |
|------|----------|------|------|
| `html` | `--font-size: 16px` | — | — |
| `h1` | `--text-2xl` | 500 (`--font-weight-medium`) | 1.5 |
| `h2` | `--text-xl` | 500 | 1.5 |
| `h3` | `--text-lg` | 500 | 1.5 |
| `h4` | `--text-base` | 500 | 1.5 |
| `label` | `--text-base` | 500 | 1.5 |
| `button` | `--text-base` | 500 | 1.5 |
| `input` | `--text-base` | 400 (`--font-weight-normal`) | 1.5 |

页面级增强样式仍可通过 Tailwind 工具类覆盖以上默认值。

### 间距系统

所有间距基于 Tailwind 间距系统，保持全站一致：

| 位置 | 间距值（Tailwind） | 像素 |
|------|-------------------|------|
| 区块垂直间距 | `py-12` | 上下各 48px |
| Hero 区域 | `py-16 md:py-20` | 上下 64px / 80px |
| 容器左右内边距 | `px-4 sm:px-6 lg:px-8` | 16px / 24px / 32px |
| 标题到内容间距 | `mt-2 ~ mt-6` | 8px ~ 24px |
| 网格间距 | `gap-4` | 16px |
| 卡片内边距 | `p-5` | 20px |
| 大面板内边距 | `p-6 md:p-8` | 24px ~ 32px |
| Footer 上边距 | `mt-20` | 80px |
| 页面最大宽度 | `max-w-[1440px]` | - |

### 圆角规范

圆角通过 CSS 变量 `--radius: 1rem`（16px） 为基数，并提供 `radius-sm` / `radius-md` / `radius-lg` / `radius-xl` 梯度。组件级别实际使用的圆角由 Tailwind 类控制：

| 组件 | 圆角值 |
|------|--------|
| 标签/小按钮 | `rounded-full` |
| 搜索栏外壳 | `rounded-[26px]` |
| 组件基础变量 | `--radius-sm`（12px）/`--radius-md`（14px）/`--radius-lg`（16px）/`--radius-xl`（20px） |

### 栅格布局系统

响应式栅格遵循以下约定：

| 场景 | 栅格配置 |
|------|----------|
| 4 列卡片网格 | `xl:grid-cols-4 md:grid-cols-2` |
| Hero 主+侧 | `lg:grid-cols-[1.1fr_440px]` |
| Footer | `lg:grid-cols-3` |

### 组件交互规范

- **悬浮效果**：卡片悬浮时 `translateY(-2px)`，阴影增强（`box-shadow` 由 `0.045` → `0.085`）
- **过渡动画**：核心组件 `160ms ~ 180ms ease`/`ease-in-out`，`card-interactive` 为 `200ms ease`
- **焦点处理**：统一 `focus-visible` 外框 `outline: 2px solid rgba(15, 23, 42, 0.16); outline-offset: 2px`
- **动效尊重**：`@media (prefers-reduced-motion: reduce)` 将所有过渡和动画时长重置为 `0.01ms`，禁用位移变换
- **按下回弹**：`:active` 状态统一 `transform: translateY(0)` 回弹

## 配色风格核心原则

使用白色中性底 + 蓝色主操作的 Apple 风方案，支持暗色模式切换。

重点不是极致设计，而是让页面具备：
- 可信度
- 产品感
- AI 行业调性
