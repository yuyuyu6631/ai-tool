# AI 工具评测平台· 验收准则

## 一、总体要求

* 面向中国 C 端用户，语言中文
* 首页风格年轻、科技感、简洁
* 功能以 Demo 演示为主，保证首页首屏、AI推荐、榜单、工具卡片、场景推荐完整可用
* 代码可运行、结构清晰、组件化
* UI与交互与 PRD/最终稿保持一致

---

## 二、代码验收准则

### 1. 前端（React / Next.js）

| 序号  | 验收项      | 详细说明                                                         |
| --- | -------- | ------------------------------------------------------------ |
| F-1 | 项目能启动    | `npm install && npm run dev` 无报错，首页可访问                       |
| F-2 | 组件化      | Navbar、Hero、ToolCard、榜单Card、场景Card、AI输入框等均独立组件               |
| F-3 | 页面路由     | 首页 `/`，榜单页 `/rank`，工具详情 `/tools/[slug]`，场景页 `/scenarios` 可访问 |
| F-4 | AI推荐接口接入 | `/api/recommend` 可以返回 2~3 个工具，前端能正确解析渲染卡片                    |
| F-5 | 响应式      | 页面在 PC 和移动端显示正常，布局不溢出，卡片自动换行                                 |
| F-6 | 状态交互     | 搜索框可输入，空输入禁用，加载状态显示，异常提示显示                                   |
| F-7 | 数据驱动     | 所有推荐模块（榜单、工具卡片、场景推荐）均可用 JSON/mock 数据驱动渲染                     |
| F-8 | 不报错      | 控制台无 React 警告或 JS 错误                                         |

---

### 2. 后端（Python / FastAPI）

| 序号  | 验收项    | 详细说明                                         |
| --- | ------ | -------------------------------------------- |
| B-1 | API可用  | `/api/recommend` 返回 JSON 格式正确，状态码 200        |
| B-2 | 响应时间   | 单次 AI推荐接口响应 < 3s（mock可放宽）                    |
| B-3 | JSON格式 | 每条推荐必须包含 `name`、`slug`、`reason`、`tags`、`url` |
| B-4 | 异常处理   | 接口超时或失败时返回统一提示 `"暂时无法获取推荐，请稍后重试"`            |
| B-5 | 缓存     | 热点推荐结果可缓存（可选，Redis/mock）                     |
| B-6 | 可扩展性   | 后端封装 AI 调用函数，可替换不同模型，不影响接口调用                 |

## 三、设计实现验收准则

| 序号   | 验收项    | 详细说明                            |
| ---- | ------ | ------------------------------- |
| D-1  | 顶部导航   | 悬浮毛玻璃效果，左Logo，中间导航项，右主按钮        |
| D-2  | Hero 区 | 主标题清晰，副标题中文、可理解、面向 C 端用户        |
| D-3  | AI输入框  | 大号、带提交按钮，输入示例 chip，空输入禁用，加载状态显示 |
| D-4  | 热门榜单   | 4张或更多卡片显示，卡片统一圆角、阴影、间距          |
| D-5  | 工具分类   | 显示首页最常用 8 个分类，带图标或颜色区分          |

| D-7  | 场景推荐   | 至少 3~5 个场景入口，每个场景可点击查看推荐工具      |
| D-8  | 文案与语言  | 全中文、口语化、清晰，标题副标题、按钮文案符合 PRD     |
| D-9  | 配色与风格  | 主色蓝紫渐变，科技感、简洁，卡片间距均匀            |

## 四、验收方式

1. **前端演示**：打开首页和各功能模块，逐条检查设计与交互效果。
2. **后端接口**：Postman 或前端测试 `/api/recommend`，检查返回数据格式、异常处理。
3. **视觉对比**：与 PRD 或最终稿对照，检查文案、颜色、间距、卡片大小、输入框样式。
4. **响应式测试**：模拟 PC、平板、手机分辨率，确保布局自适应。
5. **代码审查**：组件化结构是否合理，是否保持可维护性。
