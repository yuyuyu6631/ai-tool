# 验收标准

## 一、总体要求

- 当前验收以真实代码现状为准
- 前端、后端、测试与文档口径必须一致
- 任何“历史 Demo 路由”或“旧阶段范围”不能作为当前验收依据

## 二、前端验收标准

| 编号 | 项目 | 验收说明 |
| --- | --- | --- |
| F-1 | 项目可启动 | 根目录执行 `npm run dev` 或 `python start.py` 后，首页可访问 |
| F-2 | 主路由可访问 | `/`、`/tools/[slug]`、`/scenarios`、`/scenarios/[slug]`、`/compare/[comparisonSlug]`、`/matches`、`/auth` 可访问 |
| F-3 | 兼容路由行为正确 | `/tools` 与 `/rankings` 的重定向行为符合当前实现 |
| F-4 | 首页承接 | 首页可承接搜索、筛选、分页和视图切换 |
| F-5 | 详情页 | 工具详情页可展示基础信息、评论摘要和相似工具 |
| F-6 | 认证页 | `/auth` 提供登录、注册、已登录状态和退出能力 |
| F-7 | 后台页 | `/admin`、`/admin/tools`、`/admin/reviews`、`/admin/rankings` 前端页面存在并可在有权限时进入 |
| F-8 | 响应式 | 主要页面在桌面和移动端布局正常 |
| F-9 | 不报错 | 浏览器控制台不应出现明显运行时错误 |

## 三、后端验收标准

| 编号 | 项目 | 验收说明 |
| --- | --- | --- |
| B-1 | 目录接口 | `GET /api/tools`、`GET /api/tools/{slug}`、`GET /api/home`、`GET /api/scenarios` 正常返回 |
| B-2 | 评论接口 | 评论列表、我的评论查询与提交接口可用 |
| B-3 | 认证接口 | 注册、登录、登出、读取当前用户接口可用 |
| B-4 | 管理接口 | 后台总览、工具、评论、榜单管理接口存在 |
| B-5 | AI 能力 | `GET /api/ai-search`、`POST /api/recommend`、`POST /api/chat` 可用 |
| B-6 | 抓取/解析入口 | `POST /api/crawl/jobs`、`POST /api/parser/extract` 可调用 |
| B-7 | 数据模型 | 当前模型覆盖目录、评论、抓取、认证和后台管理所需实体 |

## 四、测试验收标准

| 编号 | 项目 | 验收说明 |
| --- | --- | --- |
| T-1 | Web 单测 | `npm run test:web` 可执行 |
| T-2 | API 测试 | `python -m pytest` 可执行 |
| T-3 | 前端 E2E | Playwright 用例覆盖 smoke、visual、navigation、auth |
| T-4 | 后台与评论测试 | 代码库中存在后台/评论相关测试 |
| T-5 | 文档一致性 | `README.md`、`doc/`、`docs/current-implementation-baseline.md` 口径一致 |

## 五、当前不作为验收前提的内容

- 旧的 `Vite` 或 `React Router` 路由实现
- 浏览器侧 `mock API`
- “后台管理未落地”的旧口径
- “评论未实现”的旧口径
- 第三方 OAuth、支付、会员、个人中心

## 六、验收方式

1. 先看 `docs/current-implementation-baseline.md`
2. 再核对真实路由、接口和模型
3. 最后运行前端与后端测试
