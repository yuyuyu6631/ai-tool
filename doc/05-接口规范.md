# 05 接口规范

## 1. 当前实现说明

本项目已彻底接入真实的 FastAPI 后端服务（`apps/api`），因此所有的 `/api/*` 请求不再是浏览器前端强行通过 `window.fetch` 拦截 Mock 假数据。
前端目前会发出真实的请求，由 Next.js 网关转发至 FastAPI 真实后端（默认 `http://localhost:8000`）。这确保前后端可以各自独立演进，接口严格遵守后端的 Pydantic Schema。

## 2. 核心大纲：AI 推荐接口

这是首页最重要的接口，用于用户的意图搜索和后端的三段式推荐响应调用。

### 2.1 接口地址

`POST /api/recommend`

### 2.2 请求头

```http
Content-Type: application/json
```

### 2.3 请求体结构

```json
{
  "query": "我想做 PPT 用什么 AI"
}
```

字段说明：
- `query`：用户输入的查询词汇。后端可能会做安全过滤或截断以预防 Prompt 注入。

## 3. 成功响应

为了最大化向后兼容我们早先组件的定义，此处直接返回“推荐结果字典数组”。响应体并非传统的 `{"code":0, "data": []}`，而是直接外抛 List 格式。

```json
[
  {
    "name": "Gamma",
    "slug": "gamma",
    "reason": "Gamma 在 PPT 制作方面表现出色，能够快速生成精美的演示文稿，大幅提升工作效率。",
    "tags": ["PPT", "演示", "自动排版"],
    "score": 9.1,
    "url": "https://gamma.app"
  }
]
```

**字段说明：**
- `name`：工具官方名称
- `slug`：唯一访问别名，用于构造前端 `/tools/gamma`
- `reason`：后端推荐模型/规则引擎产出的推荐描述
- `tags`：标签分组数组
- `score`：系统打分/热度分
- `url`：工具对应的真实域名/官网（依据验收标准必传）

## 4. 失败与异常响应

当遇到连接拥塞、模型不可调用或者解析错误时，返回下述统一的高频错误体：

```json
{
  "error": "暂时无法获取推荐，请稍后重试"
}
```
HTTP 状态码：`500`

针对非法路由，FastAPI 原生默认返回标准的 404/Method Not Allowed。

## 5. 接口维护约束

1. 后续所有的增删改查动作必须先走 `packages/contracts` 层确认类型无误，再行修改。
2. 虽然大模型容易产生幻觉或不稳定返回，但是此对内 API 的返回值结构必须保持强类型的字典队列结构，后端应当内部吞掉大语言模型的非标准返回重试机制。
3. 保证单次 AI 推荐的响应控制在可用时间内（不计算网络冷启动应尽可能 `< 3s`），后续由 Redis 介入提高命中池。
