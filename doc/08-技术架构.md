# 08 技术架构

## 1. 当前架构结论

| 层级 | 当前采用 |
| --- | --- |
| 仓库结构 | NPM Workspace Monorepo |
| 前端框架 | Next.js 15 App Router + React 19 |
| 后端框架 | FastAPI |
| 数据层 | SQLAlchemy 2 + Alembic + MySQL |
| 样式层 | Tailwind CSS 4 + `theme.css` |
| 认证 | 服务端会话 + HttpOnly Cookie |
| 测试 | Vitest + Playwright + pytest |
| Hook/文档流程 | `docs:sync` + git pre-commit 自动刷新文档基线 |

## 2. 目录分层

### 2.1 `apps/web`

职责：

- 承接公开站点和后台前端路由
- 通过 App Router 页面取数
- 通过客户端 Provider 维护登录态
- 维护页面组件、样式和前端测试

当前关键路由：

- `/`
- `/tools/[slug]`
- `/compare/[comparisonSlug]`
- `/scenarios`
- `/scenarios/[slug]`
- `/matches`
- `/auth`
- `/admin/*`

兼容路由：

- `/tools` -> `/`
- `/rankings` -> `/`

### 2.2 `apps/api`

职责：

- 提供目录、评论、认证、AI、抓取和后台接口
- 管理数据库模型与迁移
- 提供定时任务和会话认证能力

当前路由模块：

- `auth`
- `catalog`
- `reviews`
- `admin`
- `ai_search`
- `recommend`
- `chat`
- `crawl`
- `parser`

### 2.3 `packages/contracts`

职责：

- 提供共享类型与样例数据
- 作为前后端契约统一的长期收敛点

## 3. 前端架构说明

### 3.1 渲染与承接

- 首页 `/` 当前就是统一目录承接页
- `/tools` 只保留兼容性重定向
- 认证能力通过 `AppProviders -> AuthProvider` 恢复登录态
- 页面内状态主要通过 URL 查询参数驱动

### 3.2 前端能力结构

- 公开内容：首页、详情、场景、对比、匹配、认证
- 后台内容：总览、工具管理、评论管理、榜单管理
- 共享组件：Header、Footer、ToolCard、ToolReviewsPanel、AdminShell 等

### 3.3 前端测试

- 单测：Vitest
- 组件与页面测试：`src/app/**/__tests__`
- E2E：`src/e2e/*.spec.ts`

## 4. 后端架构说明

### 4.1 API 层

当前公开接口涵盖：

- 目录、分类、榜单、场景
- 评论读取与用户评论提交
- AI 搜索与推荐
- SSE Chat
- 认证
- 导入校验、抓取和解析入口
- 后台工具/评论/榜单管理

### 4.2 数据层

当前核心模型已经覆盖：

- 内容目录（支持 Head/Mainstream/Longtail 等多梯度的内容分层筛选与编辑推荐算法）
- 工具扩展元数据（`features_json`、`limitations_json`、`best_for_json`、`deal_summary`、`media_items_json`）
- 评论与评分
- 抓取与更新候选
- 用户与会话
- 管理端所需的细粒度 RBAC 权限与角色模型

### 4.3 调度与基础设施

- `apscheduler` 已接入应用生命周期
- 当前定时任务每 60 分钟触发一次抓取快照构建
- CORS 已对本地 Next.js 开发地址放开
- `/health` 与 `/health/ready` 用于存活与就绪检查

## 5. 当前架构边界

已经落地：

- 前后端分离但同仓协作
- 真实 API 与真实模型
- 细粒度 RBAC 控制与管理员权限校验
- 评论与后台管理主链路
- 自动文档基线生成
- 前端全局 404 路由防护与稳定的 E2E(Playwright) 回归测试验证

尚未完全闭环：

- 抓取审核发布仍是轻量实现
- 共享类型仍未完全收口
- OAuth、支付、个人中心未落地
